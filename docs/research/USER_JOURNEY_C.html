Audible Narrators · User Journey
Direction C — Search-First Discovery
Problem
Narrator-aware listeners are split into two types: those who know a name they want (e.g., "Jim Dale") and those who only know a feeling ("I want that big epic voice from the last fantasy book I loved"). Neither type has a supported browse path today — the nav has no Narrators entry, and a global site search for narrator names returns audiobook results, not narrator-centric pages.
Solution — Direction C
"Narrators" in the nav opens a page where a large narrator-name search field dominates the above-fold viewport — it is the first and most prominent element on the page. Below it, a horizontal scrollable "Popular Narrator" chip row surfaces known names as starting points for exploration. As the user types, a live-updating results stream appears inline — no separate results page, no button to press. For users who don't have a name, scrolling past the search field reveals the full audiobook list with filter chips appearing as a sticky bar on scroll — filter-after-browse rather than filter-then-browse.

Direction C — Search-First Discovery: The page serves both user types simultaneously — the name-knower types immediately (search field is autofocused on page load), the explorer scrolls and filters. The two paths share the same page and the same result list — no branching navigation, no second page. The "Popular Narrators" chip row acts as a bridge between the two modes: clicking a chip is a shortcut for the explorer who doesn't want to type. Platform tokens throughout: #1479cf selected state, #ffa000 amber CTA, 8px chip radius, 2px border, Arial fallback font.
