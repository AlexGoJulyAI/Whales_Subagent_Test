Direction C · User Journey · SafeChat · 2026-04-14
Category-by-Category
One category per screen — the most granular step model. Each screen focuses on a single risk topic: what it is, a real example of when it would trigger, and the three tier options. 11 category screens total before the summary review. Built for parents who want to understand each setting before committing to it.
Interaction: micro-step 13 parent screens 2 child states Pattern: quiz/survey onboarding
Flow 1 — Parent Setup
Step 1New
Parent lands on the Welcome screen
Brand, five-minute time promise, and a brief explanation that each screen will cover one topic at a time. The parent sees the total number of decisions ahead: 11 categories.
↓
Step 2New
Parent taps "Get started" — a numbered progress bar shows 11 steps
A compact progress bar at the top counts individual categories (1 of 11, 2 of 11…) rather than section groups. The parent always knows their exact position.
↓
Steps 3–13New
One category per screen — name, example, and three tier choices
Each screen shows the category name, emoji, a plain-language description, and one concrete example of when the AI would intervene. Below that: three large, tappable tier cards (Off / Balanced / Strict) with one-sentence descriptions for each. Tapping a card immediately advances to the next category.
↓
Decision point
Does the parent want to go back and change a previous selection?
If yes
They tap the back chevron. The previous category screen reloads with their prior selection still highlighted. Changing it and tapping a new tier moves them forward again.
If no
They proceed forward. Each tap advances them one category at a time until all 11 are complete.
↓
Step 14New
Summary review screen shows all 11 selections at once before saving
A compact table lists every category with the chosen tier highlighted. The parent can tap any row to jump back and change that category. When satisfied, they tap "Save settings."
↓
Step 15New
Completion screen confirms all settings are live
Green success indicator, the same 3-bullet "what happens next" list, and the locked reassurance: "Think of SafeChat as a seatbelt, not a substitute for conversation."
↓
What happens next
Parent taps Done
Settings go live immediately. The parent exits to their dashboard. Their child can begin using the AI chat.
Parent wants to revisit settings later
They return via the parent dashboard, which takes them to the summary review screen where any category can be tapped and changed individually.
Flow 2 — Child Chat Interface
Step 1New
Child opens the AI chat — same experience as Directions A and B
A "SafeChat ✓" badge in the header is the only visible indicator. The chat interface itself is identical regardless of which setup direction the parent used.
↓
Step 2New
Child sends a message — AI response is evaluated before delivery
Every reply is checked against the parent's configured tiers. This is invisible to the child and happens before the response appears.
↓
Decision point
Does the AI's response trigger any configured policy?
If no
The response appears normally. The conversation continues.
If yes
The response is blocked. A warm redirect message appears in its place.
↓
Step 3 — blocked pathNew
Child reads the redirect — empathetic, not an error
The message leads with acknowledgment, redirects to a trusted adult, and for self-harm triggers includes a crisis resource. The input field stays active so the child can keep chatting.
↓
What happens next
Child continues the conversation
They change the subject and the AI responds normally. The parent receives a background alert with context.
Child closes the app
The parent alert is sent regardless. No required action unless the parent chooses to follow up.
Direction C's distinctive move: the parent encounters each category in full focus before moving on — with a real example of what would trigger it. This maximizes informed decision-making at the cost of a longer setup time (13 screens vs. 6 in Direction A). Best suited for parents who want to understand what they are agreeing to, not just set it and forget it.
