Direction at a Glance
  • Scope B — both challenge context and attempt list visible simultaneously; the highest information density of the four options without sacrificing scan clarity
  • Inline expandable preview eliminates the click-in / click-back pattern for quick reference — the entire §2 Problem 1 behavior removed without changing the page model
  • Single search bar in the left column filters both columns — the researcher never loses challenge-level context while searching
Strategy
Direction D exists because §2 Problem 2 (no search) and §2 Problem 1 (no meaningful identity) compound each other in a specific way: the user can't find what they can't see, and the current UI forces a navigation step before anything meaningful is visible. The master-detail model solves both simultaneously — the left column provides permanent challenge orientation satisfying §4's secondary metric with no click, while the right column provides a scannable, filterable attempt list satisfying §4's primary metric. The inline preview is the decisive differentiator: it removes the click-in / click-back behavior for both the §3 researcher (quick strategy reference) and the §3 learner (outcome confirmation) without requiring a new page or modal pattern.
