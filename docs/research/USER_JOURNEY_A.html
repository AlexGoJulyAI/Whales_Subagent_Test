Audible Narrators · User Journey
Direction A — Editorial Voice
Problem
The Audible browse nav has no "Narrators" entry — listeners who love a specific voice have no direct path from global navigation to narrator-focused content. The current Popular Lists column ends at Authors with no equivalent narrator entry. A listener who wants everything narrated by Jim Dale has to search by title, read descriptions, and hope metadata surfaces the narrator — there is no curated browse path.
Solution — Direction A
"Narrators" replaces "Series" in the Popular Lists nav column. Clicking it opens a Narrators browse page where the narrator's name is the primary discovery lens — each audiobook entry leads with narrator credit above the title. A hero chip row surfaces voice-style and genre-specialty tags immediately on load; a narrator name search field sits inline with the chips. Users orient by voice first, titles second — the opposite of the default browse experience. No sidebar, no modal filter panel: all filtering happens in a single sticky chip row that collapses to a compact bar on scroll.

Direction A — Editorial Voice: The narrator name leads every list item (above title), reversing the Authors page hierarchy. The chip row is sticky on scroll and collapses to a compact summary bar ("Voice: Epic · Genre: Thriller · ×") when the user scrolls past the first screen. All filtering is chipand-search in a single row — no sidebar, no modal, no drawer. Platform token faithful: amber CTAs, #1479cf selected state, 8px chip radius, 2px border throughout.
