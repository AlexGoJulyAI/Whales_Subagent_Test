Direction A · User Journey · SafeChat · 2026-04-14
Linear Step-Through
One category group per screen. Persistent progress bar. Forward-only navigation with a back chevron available at any step. Parent completes 4 focused screens before reaching confirmation. The child's experience begins immediately after.
Interaction: step-by-step 6 parent screens 2 child states Pattern: Duolingo onboarding
Flow 1 — Parent Setup
Step 1New
Parent lands on the Welcome screen
The parent sees the SafeChat brand, a plain-language explanation of what setup involves, and a visual preview of the three tier options (Off, Balanced, Strict) before touching anything.
↓
Step 2New
Parent taps "Get started" and the progress bar appears
The first settings screen loads with a 5-dot progress bar at the top and a step counter. The parent now knows exactly how far they are in the flow at all times.
↓
Step 3New
Content settings — parent selects a tier for each of 4 categories
Self-harm, drugs, eating disorders, and illegal activities each have a three-tile selector. The Next button stays disabled until all four have a selection.
↓
Decision point
Does the parent select a tier for all 4 categories before tapping Next?
If yes
The Next button activates and the parent advances to behavioral monitoring (Step 4).
If no
The Next button stays disabled. The unselected card gets a subtle highlight so the parent knows what's missing.
↓
Step 4New
Behavioral monitoring — parent sets 3 pattern-watch categories
A blue intro card explains these settings watch trends over time, not individual messages. Parent selects tiers for loneliness, radicalization, and personal information sharing.
↓
Step 5New
AI personality — parent picks honesty level and relational style
Two independent radio-card groups: how candid the AI should be, and whether it should feel like a tool or a friend. Each group requires one selection before the Next button activates.
↓
Step 6New
Learning settings — parent configures 2 academic integrity categories
A green philosophy box frames the reason for the settings before the options appear. Parent selects tiers for homework completion detection and over-reliance pattern monitoring.
↓
Step 7New
Completion screen confirms settings are live
All 5 progress dots turn green. A success indicator, 3-bullet "what happens next" list, and the locked reassurance: "Think of SafeChat as a seatbelt, not a substitute for conversation."
↓
What happens next
Parent taps Done
Settings activate immediately. The parent exits to their dashboard. Their child can now begin using the AI chat.
Parent wants to change a setting
They tap the back chevron to return to any prior screen. Their existing selections are preserved on every screen they revisit.
Flow 2 — Child Chat Interface
Step 1New
Child opens the AI chat — sees a normal chat interface
A "SafeChat ✓" badge in the chat header signals that settings are active, but nothing else is visually different from a standard chat experience.
↓
Step 2New
Child sends a message and the AI's response is evaluated
Before the AI's reply appears, it is checked against the parent's configured settings. This happens silently in under a second.
↓
Decision point
Does the AI's response trigger any of the parent's configured policies?
If no
The AI response appears as a normal chat bubble. The conversation continues naturally.
If yes
The response is intercepted. The child sees a warm redirect message with a lock icon in place of the original reply.
↓
Step 3 — blocked pathNew
Child reads the redirect message — empathetic, not alarming
The message leads with acknowledgment ("I hear you"), redirects to a trusted adult, and for self-harm triggers includes a crisis resource. The input field stays active so the child can keep talking.
↓
What happens next
Child continues chatting
They change the subject and the AI responds normally. The parent receives a background alert about what was flagged.
Child closes the app
The parent alert is sent regardless. No immediate action is required from the parent unless they choose to follow up.
Direction A's distinctive move: the Next button is disabled until every category on the current screen has a selection — a gate that prevents a parent from accidentally skipping a setting without realizing it.
