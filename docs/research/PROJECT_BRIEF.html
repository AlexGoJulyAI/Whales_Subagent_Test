1 · Product Vision

Red Team Sample Submission is a structured learning module where users attempt to jailbreak an AI model across three increasingly difficult challenges. Each attempt is saved as a conversation. The Previous Conversations tab is the user's index of all their prior attempts — the place they return to when studying what worked, what failed, and how to improve.

Today that index is a list of timestamps. The redesign transforms it into a searchable, titled conversation library — so users can find "the political radicalization attempt" in seconds rather than opening five tabs to figure out which session was which.

2 · Problem Statements
Primary
The Previous Conversations tab surfaces zero identifying information about each conversation. Every row shows only a timestamp (e.g., "Mar 9, 2026, 9:52 AM PDT"). Two sessions from the same morning are visually indistinguishable without opening both.
Secondary
There is no search or filtering. As conversations accumulate — Challenge 1 alone has 9 attempts spanning Oct 2025–Mar 2026 — the list becomes impossible to navigate without opening every tab.
Framing note
Auto-rename is the highest-impact single fix. Full-text search delivers the most value once the list exceeds ~15 items. Both features confirmed to ship together.
3 · Target Users

Primary persona: A learner enrolled in the Red Team Sample Submission module. They are mid-journey — they have submitted attempts across multiple challenges and are returning to review past work to improve their technique.

They are research-oriented, comfortable with AI interfaces, and frustrated by the current lack of organization. Their emotional state when arriving at this tab: purposeful but impatient — they know what they're looking for but can't find it.

4 · Success Metrics
Primary metric
Time-to-locate a specific past conversation — measurably reduced. Proxy: user opens the correct conversation on the first or second click, not the fifth.
Secondary metrics
Users can identify conversations by topic without opening them; search returns the correct conversation when a keyword from the transcript is entered.
Proxy signals
Fewer "Reset Conversation" actions that appear to be "wrong conversation" corrections; reduced share-link copies that go unused.

User arrives at Previous Conversations tab — sees search bar at top + challenge accordions below
↓
2
User types keyword (e.g., "radicalization") → results filter across all three challenges
↓
3

↓
4
User clicks result card → conversation opens in new tab at /collections/{id}/feedback/{id}
Flow 2 — Browse attempts by challenge (secondary)
1
User expands a Challenge accordion → sees rich cards instead of bare timestamp rows
↓
2

↓
3
User clicks card → conversation opens in new tab with auto-generated title in the header
Flow 3 — Auto-rename on close (background, no user action)
1
User finishes a conversation and navigates away → system reads full transcript
↓
2
AI generates descriptive title (e.g., "Political Radicalization — Unsuccessful") and stores it
↓
3
Title appears on the conversation card and in the conversation view header on next load
