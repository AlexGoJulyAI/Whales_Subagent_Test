Direction A — Integrated Pills
Most native approach: safety joins the existing sub-nav as one more pill. Smallest visual footprint. Lowest learning cost.
Clean 5/5 Effective 3/5 Intuitive 4/5 Pleasing 5/5 Total 17/20
Why this direction exists
The brief's primary goal (§4) is "safety grade visible without leaving the page, within one interaction." The lowest-friction way to add a fourth filter category is to drop it into the row that already holds the type pills (Any / Home / Hotel). A user who can use those pills can use this one — no learning curve, no new visual grammar. The badge mirrors Guest Favorite's overlay placement so it reads as a peer signal, not a foreign one.

Edge Cases
Edge · Filter "Very Safe" → 0 results
Listings panel renders empty-state card: bold "No Very Safe stays match your dates" · body "Try expanding to Safe or Average to see more options." · primary CTA "Show Safe (B+)" — button style same as Filters pill, brand color. Map shows only A-zone heatmap with no markers.
Edge · Drawer open + new filter applied
Drawer auto-collapses with 150ms fade. If the same listing remains in filtered set, the badge active-state ring is preserved so user can re-tap easily. If listing is filtered out, scroll resets to top of panel.
Edge · Heatmap on + filter "All"
All four heatmap colors render simultaneously at full 0.25 alpha. Legend in bottom-left of map shows all four grade swatches. Price markers remain on top of heatmap (z-index: 2).

Backward-from-Failure — P0 Components
Safety pill: If users miss it, they will fail Task 2 (filter to Very Safe). Mitigation — when active, pill takes the full grade color so its activation state is unmissable; when default, it sits at position 4 (read-order priority) and uses the exact pill grammar of the existing type pills it lives next to.
Badge overlay: If users overlook it, they will fail Task 1 (assess safety at a glance). Mitigation — placement matches Guest Favorite (top-left of photo), so eyes already know to look there; the saturated grade color contrasts strongly against typical photography.
Inline drawer: If users don't realize the badge is tappable, they will fail Task 3 (access neighborhood detail). Mitigation — cursor pointer + hover brightness change + tooltip "Tap for safety details" on first hover per session.
Distinctiveness Log — what makes A unique
Safety filter occupies zero new vertical space — joins the existing type-pill row. B and C add structural elements; A does not.
Active filter pill takes the grade color itself. Most distinctive activation state of the three directions.
Badge is a small overlay pill on the photo top-left — mirroring Guest Favorite. B uses a metadata chip; C uses a card border.
Of the three, A is the most "native" — a user could be told this was always part of Airbnb.

Q3 Filter Placement Three candidates: (a) sub-nav pill, (b) dedicated bar, (c) Filters modal. A wins for Direction A because it preserves Airbnb's existing visual rhythm — every other filter category is a pill in this row.
Q5 Badge Style Three candidates: (a) photo overlay pill, (b) metadata chip, (c) card-border. (a) wins for Direction A because it pairs natively with Guest Favorite — same shape grammar, same position grammar.
Trade-off Acknowledged Lowest prominence of the three. Effective score 3/5 — a user who is not actively scanning for safety could miss the pill. Mitigation: active state takes the full grade color so engaged use is unmissable.

Final Falsification Pass
Failure mode: Safety pill is too subtle in default state — sits in a row of 3 other pills with identical grammar and could fail to register as "the safety filter". Users who arrived at the page knowing they want safety filtering would scan and see only "Any / Home / Hotel" until they re-read.
Mitigation: When the user has not engaged the pill, render a subtle 1px #FF385C dot inside the pill at the trailing edge — visual hint that this pill is new, removed once user has interacted with it once (sessionStorage flag).
Exact Build Instructions for Prototyper
Header height stays at 151px. Do not modify. Sub-nav row: keep the existing 3 type pills (Any / Home / Hotel), add the Safety pill at position 4 with gap: 8px. Filters button stays at far right.
Card photo: add position: absolute; top: 12px; left: 12px; badge as the FIRST child of the photo container — Guest Favorite badge sits below it if both exist. Stack order: badge (z 3) · heart (z 2) · carousel (z 1).
Inline drawer: render as a sibling of the card grid. Use CSS grid grid-row trick to make the drawer span both columns when expanded. height: auto; max-height: 0; overflow: hidden; with transition: max-height 250ms ease, opacity 200ms ease.
Map heatmap: render as divs with position: absolute and filter: blur(40px) on the map panel. Coordinates are mock — Pacific Heights top-left, Tenderloin bottom-right of a stylized SF outline. Each zone 200×200px at 0.25 alpha.
Heatmap toggle: 40×40px circular button absolute top: 14px right: 14px. State managed in React via useState. When toggled on, heatmap zones get opacity: 1 from 0; transition opacity 200ms ease.
Mock SF data — Pacific Heights A · Noe Valley A · Mission B · Castro B · SoMa C · Outer Sunset C · Tenderloin D · Civic Center D. Use these eight neighborhoods for all listing cards. Cycle through.
Font fallback chain (Airbnb Cereal VF unavailable): "Circular", -apple-system, "system-ui", Roboto, "Helvetica Neue", sans-serif. Do not use system serif fallback.
