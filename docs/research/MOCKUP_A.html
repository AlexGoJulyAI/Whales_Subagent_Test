Industrial / utilitarian · Three always-visible panes — no expand interactions
Goal thread: The three-pane model eliminates all expand-click interactions (§2 Problem 3) and makes AI topic names the primary scannable element in the center pane (§2 Problem 1), directly serving the §4 primary metric of locating a conversation in under 10 seconds.
What this changes
1
Challenge pane replaces the challenge cards + accordion — Challenges become a slim persistent column on the left; one click activates, no expand/collapse toggle.
2
AI topic name as primary row label in center pane — AI-generated topic name (e.g. "Suicidal Ideation Loop") is the dominant, bold text. Date is de-emphasized below it.
3
Search bar at top of center pane filters attempts — Scoped to the active challenge; instant filter by topic name, date, or badge value.
4
Right pane shows inline preview (no new tab on first click) — First click loads the first 2–3 messages + result badge without leaving the page.
5
"Open full view →" button in right pane opens feedback tab — Second-action button for users who need to complete scoring or review the full transcript.

Existing Screen Analysis
Current UI — Previous Conversations page
  • 1
    Date-only labels: Each conversation row shows only a date (e.g. "Apr 12, 2026") as its primary identifier — there is no AI-generated topic name, so users cannot distinguish one attempt from another without clicking into every row.
  • 2
    No search: The page has no filter or search input, forcing users to scan the full list linearly; with many attempts this becomes unusable at scale.
  • 3
    Separate accordion per challenge: Each challenge is a collapsible card that must be manually expanded; users cannot see all challenges' attempts simultaneously, creating extra click-overhead to compare across challenges.
