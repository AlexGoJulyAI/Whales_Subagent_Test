Goal thread: Direction A converts parents by removing every trace of "settings UI" — each decision is a plain question they already know how to answer. §4 primary metric (>80% completion) is directly served by this one-question-at-a-time model: parents who commit to answering question 10 almost never abandon before 21. §3 identifies the parent as non-technical and cautious; this format meets them exactly where they are.
What this direction does
  • ① Question format — Each safety setting is a full-screen question in plain English instead of a settings row, removing technical framing for non-expert parents.
  • ② Progress counter — A persistent "Question 7 of 21" label and thin progress bar give parents a precise sense of how close they are to done.
  • ③ Off/Balanced/Strict pills — Three answer pills replace toggles; the selected state fills with the severity color (gray/amber/red) so the choice is impossible to miss.
  • ④ Alert card — Left-border severity coding on each dashboard card communicates urgency before the parent reads a word.
  • ⑤ Chat bubbles — Parent and child share the same SF Pro + forest-green token set, unified visual language across both user types.

Direction at a Glance
  • One safety setting = one full-screen question in plain English
  • Off/Balanced/Strict answers are pill buttons, not toggle rows — tapping fills with severity color

Strategy

Direction A exists because §3 identifies the parent as non-technical and potentially overwhelmed by AI terminology. The question-per-screen format removes the word "settings" entirely — replacing it with decisions parents already know how to make intuitively. §4's primary metric (>80% completion) is served by removing every reason to stall: there is nothing to scroll, nothing to compare, and nothing to configure — only a question to answer. Parents who commit to answering question 10 almost never abandon before 21.

Above the Fold
  1. Question text — earns first position because it IS the complete task for this screen
  2. Example — contextualizes what "balanced" means without requiring AI expertise
  3. Pill answers — decision affordances positioned after the user has read the question

Design rationale

The left-border severity stripe is a pre-attentive visual cue — parents see red before they read the card title. This directly reduces cognitive load: the most urgent item is immediately obvious without scanning text. Unread cards use Semibold weight; read cards use Regular — a subtle but consistent signal of information status carried from iOS Mail conventions.
