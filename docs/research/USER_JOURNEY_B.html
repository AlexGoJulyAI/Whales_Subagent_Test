Audible Narrators · User Journey
Direction B — Faceted Browse
Problem
There is no entry point in the Audible browse navigation for narrator-centric discovery. Listeners who want to explore by voice style, language, or genre specialty have no structured filter path — they must either know a narrator's name in advance or scroll through undifferentiated results hoping narrator metadata surfaces. Power users — those most likely to browse by narrator — have the least support for their browsing model.
Solution — Direction B
"Narrators" enters the Popular Lists nav, opening a two-column browse page: a persistent left filter panel (narrator name search + genre specialty + voice style + language + most-listened sort) and a dense results column on the right. Both panels are independently scrollable. Filter selections are additive and visible as removable tags above the results. The layout is borrowed from the mental model of advanced search — power users recognize it immediately, and the full filter set is always visible without any interaction to reveal it.

Direction B — Faceted Browse: A persistent two-column layout where all filter dimensions are visible on load — no chip row that requires guessing what filters exist. The left panel is fixed-width (240px), independently scrollable, and always present on desktop. On mobile (390px), the filter panel collapses behind a "Filters" button that opens a full-screen drawer. Platform tokens throughout: #1479cf selected chips, #ffa000 amber CTAs, 8px chip radius.
