Direction B · User Journey · SafeChat · 2026-04-14
Single Page Review
All 4 setting sections on one scrollable screen, organized under sticky section headers. The parent scrolls at their own pace, configures in any order, and taps "Save settings" at the bottom when ready. No enforced step sequence.
Interaction: free-scroll 2 parent screens 2 child states Pattern: settings page
Flow 1 — Parent Setup
Step 1New
Parent lands on the Welcome screen
Same welcome as Direction A: brand, five-minute time promise, tier overview (Off / Balanced / Strict). No progress bar here since there is only one configuration screen ahead.
↓
Step 2New
Parent taps "Get started" and arrives on a single long configuration page
All 4 setting sections appear in one scrollable view: Content, Behavioral, Personality, and Learning. Section headers are sticky so the parent always knows which section they are in.
↓
Step 3New
Parent reviews and configures sections in any order they choose
Each category card shows its tier selector inline. The parent can scroll up and down freely, change any selection at any time, and take a break mid-way without losing their progress.
↓
Decision point
Has the parent configured every required category before tapping Save?
If yes
The "Save settings" button at the bottom of the page is active. The parent taps it and moves to the completion screen.
If no
The Save button is disabled. A small counter near the button shows how many categories still need a selection (e.g. "3 remaining").
↓
Step 4New
Parent taps "Save settings" — completion screen confirms everything is live
A full-screen success state: green checkmark, settings summary (one line per section showing the choices made), and the reassurance: "You can change any of these anytime."
↓
What happens next
Parent taps Done
Settings go live. The parent exits to their dashboard. The child can begin using the AI chat interface.
Parent wants to re-configure
They tap "Edit settings" on the completion screen and return to the same single-page configuration view with all their previous choices preserved.
Flow 2 — Child Chat Interface
Step 1New
Child opens the AI chat — no visible difference from a normal chat
A small "SafeChat ✓" badge in the chat header is the only indicator that safety settings are running. The child's experience is otherwise identical to any chat app.
↓
Step 2New
Child sends a message — the AI response is evaluated before delivery
The AI generates a reply, which is checked against the parent's settings before appearing on screen. This happens silently and quickly.
↓
Decision point
Does the AI's response trigger any configured policy?
If no
The response appears normally. The conversation continues.
If yes
The response is blocked. A warm redirect message appears in its place.
↓
Step 3 — blocked pathNew
Child reads the redirect — empathetic tone, not an error message
The message leads with acknowledgment, redirects to a trusted adult, and for self-harm triggers includes a crisis resource strip. The input field stays active.
↓
What happens next
Child continues the conversation
They move to a new topic. The AI responds normally. The parent receives a background alert.
Child closes the app
The parent alert is sent regardless. No required action unless the parent chooses to follow up.
Direction B's distinctive move: the parent configures everything in one continuous scroll, in any order they choose — suitable for parents who want a full overview before committing to any selections. The tradeoff is that it places all 11 settings on one screen, which may feel heavier than Direction A's step-by-step approach.
