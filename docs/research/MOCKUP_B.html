Direction B — Prominent Bar
Highest visibility: dedicated horizontal safety bar between search and listings. Safety becomes a first-class peer of search, not a sub-filter.
Clean 3/5 Effective 5/5 Intuitive 5/5 Pleasing 4/5 Total 17/20
Why this direction exists
The brief's §3 Problem 1 calls out "no safety filter exists in the filter bar." Direction A solves this minimally; Direction B solves it loudly. For a user who landed on the page specifically because they care about safety, the bar removes any chance of missing the feature: it sits in their reading path, full-width, with the four tiers always visible. The trade-off is 56px of new vertical real estate above the listings.

Edge Cases
Edge · Filter "Very Safe" → 0 results
Listings panel renders empty-state at full width below the bar: bold "No Very Safe stays match your filters" · body "Try Safe to see {N} more options." · CTA "Show Safe (B+)" — same brand-pill style. Result summary in bar updates to "0 Very Safe stays" in #C0292B color for emphasis.
Edge · Drawer open + new tier selected
Drawer auto-collapses with 150ms fade. Bar updates result summary. If the previously-tapped listing remains in the new filter set, no scroll change. If filtered out, scroll resets to top of panel (smooth, 300ms).
Edge · Heatmap on + bar at "All"
All four heatmap colors render at full 0.25 alpha. Map legend visible. Bar shows "{N} stays · all tiers" in result summary.

Backward-from-Failure — P0 Components
Safety bar: If users miss it, they have failed not just Task 2 but the entire feature — the bar IS the feature in B. Mitigation — full-width placement directly in scroll path between header and first card row makes missing it physically impossible. Sticky behavior keeps it on screen during long scrolls.
Grade chip: If users overlook it on cards, they will fail Task 1 (per-card assessment). Mitigation — saturated grade-color fill placed in the same row as star rating; eye-tracking shows users always check the meta row.
Inline drawer: Same as Direction A — cursor + hover + first-time tooltip.
Distinctiveness Log — what makes B unique
Safety gets its own horizontal zone. A keeps it inline, C hides it in a modal. B owns 56px of vertical real estate.
Tier control is a segmented control with all 4 tiers visible — A uses a single pill that opens a dropdown, C uses checkboxes inside a modal.
Badge is a letter-grade chip in the metadata row — sits next to star rating as a peer. A overlays the photo; C uses a card border.
Bar shows a live result summary — "12 Very Safe stays" — neither A nor C surface this count proactively.
Designer Notes for Prototyper · Direction B
Internal Design Debate Summary
Q3 Filter Placement Three candidates: (a) sub-nav pill, (b) dedicated bar, (c) Filters modal. (b) wins for B because the brief's primary goal demands "within one interaction" — a segmented control with all tiers visible is one visual scan + one click. Zero discovery cost.
Q5 Badge Style Three candidates: (a) photo overlay, (b) metadata chip, (c) card border. (b) wins because B's bar already commits to a "loud, declarative" register — a peer-of-star-rating chip continues that register; a photo overlay would compete with Guest Favorite.
Trade-off Acknowledged Lowest cleanliness score of the three (3/5) — adds 56px of new UI vertical, real cost on a 1440px design that's already information-dense. Justified by Effective + Intuitive both at 5/5.

Final Falsification Pass
Failure mode: Bar feels heavy and slows perceived page load — Airbnb users come for housing, not safety, and a 56px-tall bar above the listings could read as a banner ad on first impression. If a user dismisses the bar mentally before reading it, the feature is invisible.
Mitigation: Use a subtle gradient (linear-gradient(90deg, #FFFFFF, #FAFAFA)) rather than a colored bar — keeps the bar from reading as promotional. Border-top + border-bottom (1px #DDDDDD) ties the bar visually to the existing search-form rhythm. Grade-color dots inside the segments provide just enough chromatic interest to signal "informative" without screaming.
Exact Build Instructions for Prototyper
Insert <SafetyBar /> as a sibling of the listings panel, between the sticky header and the .listings-grid. Bar must be sticky with top: 151px so it pins below the header on scroll.
Segmented control: 4 buttons in a flex row inside a single rounded container. Use a single div.active-fill as an absolute-positioned inner element that animates translateX + width when selection changes — gives the fluid "iOS segmented control" feel.
Result summary: bind to React state. Format "{count} {tier} stays". When count is 0, color text #C0292B and add the empty-state CTA inline below the listings panel.
Card grade chip: add to the metadata row as the LAST flex child with margin-left: auto. The star rating moves slightly left to make room. Star rating + chip share row height 22px.
Heatmap toggle auto-syncs: when bar selection !== "All", set heatmapOn = true via effect. Allow manual override via direct toggle click.
Mock SF data — same 8 neighborhoods: Pacific Heights A · Noe Valley A · Mission B · Castro B · SoMa C · Outer Sunset C · Tenderloin D · Civic Center D.
Font fallback chain: "Circular", -apple-system, "system-ui", Roboto, "Helvetica Neue", sans-serif.
