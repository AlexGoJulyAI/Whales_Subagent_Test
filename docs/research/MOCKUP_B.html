Goal thread: Direction B gives parents a map before they start — the hub screen surfaces all 5 categories simultaneously so §4’s completion metric is served by visible progress, not blind faith. §3 identifies parents as cautious: seeing 3 of 5 cards already green reduces anxiety better than a progress bar ever could. The hub doubles as the settings editor, meaning there is zero re-learning cost when a parent returns months later.
What this direction does
  • 1 Category hub cards — Five cards replace a wizard — parents see the full map, pick any order, and watch each card fill green when done.
  • 2 Completion status badge — Each card shows “✓ Configured” in forest green or “Not configured” in gray — progress is visible at all times.
  • 3 Category detail list — Inside each card, subcategories are grouped cells with Off/Balanced/Strict pill rows — scrollable, dense, fast to complete.
  • 4 Alert card with edit link — Dashboard alert cards link directly to the relevant hub card, so a parent can adjust a setting without searching.
  • 5 Chat bubbles — Same SF Pro + forest-green token set as parent view — unified language across both users.

Strategy

Direction B exists because §3 identifies the parent’s key anxiety as “am I almost done?” — a question a linear wizard can never answer until you’re actually done. The hub model pre-answers that question before the first tap: parents see the entire scope of the task mapped as 5 cards, choose where to start, and watch progress fill in as they go.

§4’s >80% completion metric is served by making partial completion visible and persistent: a parent who has 3 of 5 green is far more likely to complete than one who can’t tell if there are 2 or 12 steps remaining.
