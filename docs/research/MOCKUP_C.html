Goal thread: Direction C anchors setup in an emotional outcome rather than a technical task. Showing the child's name as the dashboard hero before any configuration creates personal investment that drives §4's completion metric more reliably than progress bars. §3 identifies the parent's emotional state as "cautious and wanting reassurance" — seeing their child's name already in the product IS the reassurance. Setup happens in a sheet; the destination never disappears.
What This Direction Does
  • 1 Child name hero — The parent's first view after signup is their child's name in large display type — making the entire setup feel personal before a single setting is touched.
  • 2 Setup overlay banner — A forest-green banner sits over the empty dashboard: "SafeChat is almost ready for Aria" with a "Set up now" CTA — explicit, can't be missed.
  • 3 Configuration sheet — All five categories live in one bottom sheet with a tab strip at top — parent taps "Next" to advance sections, never leaves the dashboard.
  • 4 Alert detail sheet — Tapping an alert opens a sheet with an inline "Edit setting" action — parent can change a setting without ever navigating away from the dashboard.
  • 5 Chat bubbles — Same SF Pro + forest-green tokens as parent view — unified language across both users.

Direction at a Glance
  • Parent sees their child's name in the app before any configuration — setup feels personal, not procedural
  • The setup banner sits over the destination (dashboard), not in front of a form — parent can see what they're building
  • When setup is done, the sheet dismisses and the parent is already on their dashboard — no redirect needed
Strategy

Direction C exists because §3 describes the parent's emotional state as cautious and wanting reassurance. The strongest reassurance is not a progress bar or explanatory copy — it is seeing their child's name in the product before they've done a single thing. This creates personal investment that sustains motivation through the 21-setting configuration. §4's primary metric (>80% completion) is served by emotional anchoring: parents who are configuring for a specific real person they love are far less likely to abandon than parents configuring abstract policies. The sheet model ensures the destination is always visible — setup never feels like a detour.

Above the Fold
  • 1st: Child's name (Aria) at 40px — earns first position as the emotional anchor for the entire session
  • 2nd: Setup banner — explicit CTA, but framed as completing something personal, not a required step
  • 3rd: Empty alert feed — shows the parent what they'll be monitoring; builds anticipation
